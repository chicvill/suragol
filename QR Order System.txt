나는 왕궁중화요리라는 식당에서 사용할 주문 앱을 개발한다.
고객이 테이블에 부착된 QR 코드로 메뉴판에 접속하여 주문한다.
주문 내역은 카운터와 주방의 태블릿에 주문 내역이 전달된다.
테이블의 탭에는 주문자가 앉은 테이블 번호와 주문내역이 표시되어 결제를 기다린다.
테이블 번호와 주문내역이 표시되고, 이를 참조하여 음식을 조리한 뒤 완료 버튼을 눌러 상태를 카운터와 주문번호 게시용 탭에 알린다.
주문번호 게시용 탭은 웹서버 기능과 조리 완료 시 주문번호를 게시한다.
식사 후 고객이 결제하면 카운터에서 결제 완료 버튼을 눌러 처리를 종료한다.
python, javascript, html 등을 이용하여 가장 간단하게 코딩한다.

 왕궁중화요리 주문 앱 개발 완료 보고서
중식당 "왕궁중화요리"를 위한 실시간 주문 및 관리 시스템 개발이 완료되었습니다. 이 시스템은 고객의 핸드폰 접속을 지원하며, MQTT 프로토콜을 사용하여 모든 기기 간의 실시간 동기화를 보장합니다.

구현된 주요 기능
1. 고객용 모바일 메뉴 (Customer View)
접속 방식: 각 테이블에 비치된 QR 코드를 통해 http://[서버IP]:8888/customer/[테이블번호] 주소로 접속합니다.
기능:
메뉴 카테고리(면류, 밥류, 만두류 등)별 탐색.
장바구니 담기 및 실시간 주문 전송.
장바구니 상세 관리: 주문 전 장바구니에서 각 항목의 수량을 추가/감소시키거나 항목을 삭제하는 기능 추가.
현대적인 다크 모드 기반의 프리미엄 디자인 적용.
2. 주방용 실시간 대시보드 (Kitchen View)
주소: /kitchen
기능:
조리 완료 처리: 주방에서 조리 완료 버튼 클릭 시 즉시 리스트에서 제거되고 실시간으로 게시판과 카운터에 상태가 전송됩니다.
새로운 주문 발생 시 즉시 리스트에 추가 (새로고침 불요).
3. 카운터용 결제 관리 (Counter View)
주소: /counter
기능:
테이블별 주문 합계 금액 자동 계산.
조리 완료 상태 실시간 확인.
고객 식사 후 '결제 완료' 처리 시 모든 시스템에서 해당 테이블 데이터 정리.
4. 주문 번호 표시판 (Display View)
주소: /display
기능:
주방에서 조리가 완료된 테이블 번호를 대형 화면에 크게 표시.
새로운 번호 등장 시 알림음(Chime) 재생.
기술적 특징
실시간 통신 (Socket.IO): 매장 내 서버를 통해 고객 핸드폰과 태블릿들이 끊김 없이 데이터를 주고받습니다.
안정적인 데이터 로딩: 페이지를 새로고침 하더라도 서버(app.py)에서 현재 활성화된 주문 정보를 다시 가져와 상태를 유지합니다.
가장 간단한 코딩: SQLite 데이터베이스를 사용하여 주문 정보를 안전하게 보관합니다.
사용 방법
로컬 실행 (가상환경): 루트 폴더의 run.bat 파일을 더블 클릭합니다.
도커 실행 (컨테이너): 루트 폴더의 docker-run.bat 파일을 실행합니다.
팁: 도커가 켜져 있지 않으면 친절한 안내 메시지가 나타납니다.
수동 도커 빌드: docker build -t wang-gung-app . 명령어로 이미지를 직접 빌드할 수 있습니다.
관리자 접속: 브라우저에서 http://localhost:8888에 접속하여 각 대시보드로 이동합니다.
주문 테스트: 고객용 페이지에서 주문을 넣고, 주방 -> 카운터로 이어지는 흐름을 확인하세요.
시연 영상 (조리 완료 및 실시간 동기화)
주방에서 조리 완료 처리 시 게시판(Display)과 카운터(Counter)가 실시간으로 연동되는 과정을 담은 영상입니다.
